Get started with deep learning OCR

A practical introduction to modern OCR with a co-published repository including everything needed for training and testing.

Aki Kutvonen

·

Published in

Towards Data Science

·
7 min read
·
Apr 30, 2020

A while ago I started investigating potential OCR implementations as a goal to create a clear Convolutional Recurrent Neural Network (CRNN) baseline in tf.keras 2 using only generated training data. An additional goal was to create a repository which would allow exploring different architectures and data domains later using the same template. The repository should be modular, easy to use and filled with explanatory comments so that my coworkers could follow easily.

After some high intensity Googling, I failed to find a repository which fulfilled my requirements. While there are some good resources and implementations out there, I decided to create my own [1, 2, 3].

Conventional document processing pipelines function by applying OCR software to parse text from images of text. Typically the images are prepared in advance by bounding box modules which isolate regions of text within the image for the OCR module. Later stages of the pipeline will then: parse the text, extract meanings, key words, values and tables and return results in a more programmatically ingestible form, such as: database entries, CSVs, or integrate the outputs with the company’s internal systems.

In this post, the focus is on the OCR phase using a deep learning based CRNN architecture as an example. A complete, functioning implementation is co-published in GitHub and is meant to serve as a template end-to-end pipeline including data generation and inference. The focus has been on providing a clear and well-documented pipeline which could be easily extended to different architectures, not on the neural network (NN) architecture itself.

My aim is for people to be able to train and test a simple OCR model without access to GPUs or any training data using only the instructions in the repository. This post will go into some of the implementation details, but those are not really needed for those just wanting to try out the code.
